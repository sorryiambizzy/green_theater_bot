# Rutracker Telegram Bot — Design

**Date:** 2026-06-24

## Overview

Telegram-бот для поиска торрентов на rutracker.org. Нужен, потому что rutracker заблокирован в России; бот крутится на VPS за рубежом, где сайт доступен напрямую. Любой пользователь может найти раздачу и получить .torrent файл + magnet-ссылку, не включая VPN.

## Stack

- Python 3.11+
- `python-telegram-bot` — Telegram Bot API
- `requests` + `BeautifulSoup4` — скрапинг rutracker
- `python-dotenv` — конфигурация из `.env`

## Структура проекта

```
rutracker-bot/
├── bot.py          # точка входа, обработчики команд Telegram
├── rutracker.py    # скрапер: логин, поиск, скачивание .torrent
├── config.py       # загрузка настроек из .env
├── .env            # секреты (не коммитить)
├── .env.example    # шаблон для деплоя
└── requirements.txt
```

## Конфигурация (.env)

```
TELEGRAM_BOT_TOKEN=...
RUTRACKER_USERNAME=...
RUTRACKER_PASSWORD=...
```

Аккаунт на rutracker — отдельный технический, созданный специально под бота.

## Команды бота

- `/start` — приветственное сообщение с инструкцией: "Отправь мне название фильма, сериала или игры — найду торрент на rutracker"
- Любой другой текст — воспринимается как поисковый запрос

## Пользовательский сценарий

1. Пользователь отправляет боту текст с поисковым запросом (например: `Интерстеллар 2014`)
2. Бот отвечает `Ищу...`
3. Скрапер логинится на rutracker (или использует существующую сессию) и выполняет поиск
4. Бот отображает до 10 результатов в виде inline-кнопок: `[Название | Размер | Сиды]`
5. Пользователь нажимает на нужный результат
6. Бот скачивает .torrent файл и извлекает magnet-ссылку со страницы раздачи
7. Бот отправляет пользователю .torrent файл и magnet-ссылку текстом

## Модуль rutracker.py

- `login()` — POST-запрос авторизации, сохраняет сессию (`requests.Session`)
- `search(query)` → список результатов `[{id, title, size, seeders}]`
- `get_torrent(topic_id)` → `(bytes, magnet_link)` — скачивает .torrent, парсит magnet

Сессия создаётся один раз при старте. При получении статуса 302 на страницу логина — автоматический перелогин.

**URL-ы rutracker:**
- Логин: `https://rutracker.org/forum/login.php` (POST)
- Поиск: `https://rutracker.org/forum/tracker.php?nm={query}` (GET)
- Скачивание: `https://rutracker.org/forum/dl.php?t={topic_id}` (GET)
- Страница раздачи: `https://rutracker.org/forum/viewtopic.php?t={topic_id}` (GET, для magnet)

## Обработка ошибок

| Ситуация | Ответ пользователю |
|---|---|
| Rutracker недоступен | `Rutracker временно недоступен` |
| Ничего не найдено | `По запросу ничего не найдено` |
| Сессия истекла | Автоматический перелогин, пользователь не замечает |
| CAPTCHA при логине | Требует ручного вмешательства (редкий кейс) |

Файлы .torrent весят 10–200 KB — лимит Telegram в 50 MB не является ограничением.

## Деплой

- VPS за рубежом (Hetzner, DigitalOcean и т.п.) — rutracker доступен без VPN
- Запуск как `systemd`-сервис с автостартом
- Секреты в `.env` файле на сервере (не в репозитории)
- Бот stateless — база данных не нужна

## Доступ

Бот открыт для всех пользователей Telegram без ограничений.
